## Contributing to Documentation

When adding new documentation:

1. **Choose the right location:**
   - Current features → `docs/current/`
   - API docs → `docs/current/api/`
   - Guides → `docs/current/guides/`
   - Security → `docs/current/security/`
   - Testing → `docs/current/testing/`

2. **Follow naming conventions:**
   - Use descriptive names
   - Include feature/system name
   - Example: `QUICK_START_FEATURE_NAME.md`

3. **Update this index:**
   - Add link in appropriate section
   - Include brief description

4. **Archive old docs:**
   - Move superseded docs to `history/archive/`
   - Update references
