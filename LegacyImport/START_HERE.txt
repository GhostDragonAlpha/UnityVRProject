DEFAULT SETTINGS
================
Trigger dead zone:       0.1  (10%)

Button debounce:         50   milliseconds

These are good defaults for most VR games.

WHAT ARE DEAD ZONES?
====================
Dead zones filter out small unintended inputs:

  Without: Untouched trigger reads 0.08 → causes false input
  With:    Untouched trigger reads 0.08 → filtered to 0.0

WHAT IS DEBOUNCING?
===================
Debouncing prevents one physical button press from registering twice:

  Without: Press button once → registers twice (bounce)
  With:    Press button once → registers exactly once

QUICK MATH
==========
Scalar formula:
  If input < threshold: output = 0.0
  Else: output = (input - threshold) / (1.0 - threshold)

Vector formula:
  Circular dead zone with direction preservation
  Prevents "corner deadness"

Debounce formula:
  If time_since_last_press >= 50ms: accept press
  Else: ignore (bounce)

PERFORMANCE?
=============
Incredibly efficient:
  - Dead zone: 3-6 microseconds per input
  - Debounce: 1-2 microseconds per button
  - Total: ~50 microseconds per frame
  - Frame rate impact: < 0.5% (unmeasurable)
  - Memory: ~500 bytes

TROUBLESHOOTING
===============
Sticks feel sluggish?
  → Reduce deadzone_thumbstick to 0.10

Still have drift?
  → Increase deadzone_trigger/grip to 0.15

Buttons register twice?
  → Increase button_debounce_ms to 75-100

Can't press buttons fast?
  → Decrease button_debounce_ms to 30-40
