WORKFLOW OVERVIEW
================================================================================

Step 1: Export Production Build
--------------------------------
Windows:
  cd C:\godot
  export_production_build.bat

Linux/Mac/Bash:
  cd /c/godot
  ./export_production_build.sh

What it does:
  - Locates Godot executable
  - Verifies export preset configuration
  - Backs up previous build
  - Exports with release optimization
  - Validates build artifacts
  - Generates SHA256 checksums
  - Creates build metadata

Expected output:
  - build/SpaceTime.exe (~93 MB)
  - build/SpaceTime.pck (~0.15 MB)
  - build/SpaceTime.exe.sha256
  - build/SpaceTime.pck.sha256
  - build/BUILD_INFO.txt
  - build/export_log_*.txt

Duration: 1-3 minutes

Step 2: Validate Build
-----------------------
Command:
  cd C:\godot
  python validate_build.py

What it validates:
  ✓ Executable exists and has correct format (PE, x86_64)
  ✓ PCK exists or is embedded in executable
  ✓ File sizes are reasonable (> 10 MB for EXE)
  ✓ Checksums match stored values
  ✓ Export log has no critical errors
  ✓ Dependencies are present (if needed)

Expected output:
  - build/VALIDATION_REPORT.txt
  - Console output with pass/fail results
  - Exit code 0 = pass, 1 = fail

Duration: < 10 seconds

Step 3: Test Exported Build
----------------------------
Windows:
  cd C:\godot
  test_exported_build.bat

Linux/Mac/Bash:
  cd /c/godot
  ./test_exported_build.sh

What it tests:
  ✓ Application launches successfully
  ✓ No crash on startup
  ✓ Scene loads correctly
  ✓ HTTP API responds on port 8080
  ✓ Autoloads initialize properly
  ✓ VR system initializes (if headset connected)
  ✓ Application closes cleanly

Manual verification required:
  - Visual inspection of application window
  - HTTP API endpoint testing with curl
  - Console output review

Duration: 5-10 minutes (manual testing)

Step 4: Package for Deployment
-------------------------------
Windows:
  cd C:\godot
  package_for_deployment.bat

Linux/Mac/Bash:
  cd /c/godot
  ./package_for_deployment.sh

What it creates:
  - deploy/SpaceTime-Release-v1.0/ directory with:

    - LICENSE.txt
    - DEPLOYMENT_INFO.txt
  - deploy/SpaceTime-Release-v1.0.zip (compressed archive)
  - deploy/SpaceTime-Release-v1.0.zip.sha256

Duration: < 1 minute

API SERVICES
================================================================================

The exported build includes these services:

HTTP API:
  Port: 8080 (configurable via SPACETIME_HTTP_API_PORT)
  Protocol: HTTP REST
  Endpoints: /status, /state/scene, /state/player, etc.
  Authentication: Basic (token-based)

WebSocket Telemetry:
  Port: 8081 (configurable via SPACETIME_TELEMETRY_PORT)
  Protocol: WebSocket
  Data: Real-time performance metrics, events
  Compression: GZIP (for payloads > 1KB)

Service Discovery:
  Port: 8087
  Protocol: UDP broadcast
  Purpose: Network service announcement

Note: The GodotBridge (legacy port 8082) is DISABLED in exports.
Use HttpApiServer (port 8080) instead.

TROUBLESHOOTING QUICK REFERENCE
================================================================================

Export fails:
  → Check Godot executable path in script
  → Verify export templates are installed
  → Review export_log_*.txt for errors
  → Try manual export from Godot Editor

Validation fails:
  → Check executable size (should be > 10 MB)
  → Verify PCK exists or is embedded
  → Review VALIDATION_REPORT.txt
  → Re-run export if files are corrupted

Build won't launch:
  → Install Visual C++ Redistributable (x64)
  → Update graphics drivers
  → Check Windows Event Viewer
  → Run with SPACETIME_DEBUG=1 for verbose logs

HTTP API not responding:
  → Wait 15 seconds after launch for initialization
  → Check if port 8080 is available
  → Verify firewall settings
  → Test with: curl http://127.0.0.1:8080/status

VR not working:
  → Install OpenXR runtime (SteamVR, Oculus, etc.)
  → Connect headset before launching
  → Update headset firmware
  → Check project.godot: xr/openxr/enabled=true

Low performance:
  → Update graphics drivers
  → Close background applications
  → Lower quality settings
  → For VR: Match headset refresh rate (90 FPS target)
